## 1. Project Overview

A modern cooking blog built with Next.js (App Router), Tailwind CSS v4, shadcn/ui, deployed to Cloudflare Pages via `@opennextjs/cloudflare`. Content is authored in MDX (recipes) and Markdown (blog posts) with gray-matter frontmatter. Nutritional info is automatically fetched via the Edamam Nutrition Analysis API.

## 4. Critical Rules & Constraints

### 🔴 CRITICAL: Cloudflare Workers have NO `fs` at runtime
- All filesystem reads (`readdirSync`, `readFileSync`) happen in `lib/recipes.ts` and `lib/posts.ts`
- These are ONLY safe because every route using them calls **`generateStaticParams()`** to pre-render at build time
- NEVER add a route that reads content at request time — it will crash on Cloudflare

### 🔴 CRITICAL: `params` is a Promise in Next.js 15+
```typescript
// ✅ CORRECT — always await params
export default async function Page({ params }: { params: Promise<{ slug: string }> }) {
  const { slug } = await params;
}

// ✅ CORRECT — always await in generateMetadata too
export async function generateMetadata({ params }: { params: Promise<{ slug: string }> }) {
  const { slug } = await params;
}
```

### 🔴 CRITICAL: Tailwind v4 uses CSS `@theme`, NOT `tailwind.config.*`
- All theme configuration goes in `src/app/globals.css` using the `@theme inline {}` directive
- `tailwind.config.*` files (v3 style) are ignored — do not add config there
- Use `@plugin "@tailwindcss/typography"` in CSS, not plugins in config

### 🔴 CRITICAL: shadcn/ui Radix Luma style
- Uses `data-slot` attributes and does NOT use `React.forwardRef`
- Components generated by `npx shadcn@latest add [component]` will use this style automatically
- Old-style shadcn components (forwardRef-based) are incompatible

### 🔴 CRITICAL: Post frontmatter MUST use `excerpt` not `description`
- The `recipes.ts` and `posts.ts` libs read `data.excerpt`
- MD files using `description:` will silently return `undefined` excerpt

### 🟡 Content is in `content/` not `src/content/`
- Recipe files: `content/recipes/*.mdx`
- Blog post files: `content/posts/*.md`
- The lib paths must mirror this

### 🟡 No `dangerouslySetInnerHTML` for markdown
- Use `react-markdown` for safe rendering
- Never use `dangerouslySetInnerHTML={{ __html: post.content }}`

### 🟡 NPM naming restriction
- Package name in `package.json` must be lowercase (e.g., `"name": "appropriated-kitchen"`)
- Project directory can have capitals, but NPM name cannot

## 7. Design System

### Colors (defined in `globals.css` `@theme inline`)
- `--background`: Warm off-white (`oklch(0.985 0.003 75)`)
- `--foreground`: Deep navy-charcoal (`oklch(0.13 0.02 260)`)
- `--primary`: Vibrant indigo (`oklch(0.45 0.24 280)`)
- `--accent`: Warm coral (`oklch(0.65 0.22 20)`)
- `--muted`: Subtle gray-blue
- Dark mode variants are defined in `.dark { ... }`

### Fonts (via `next/font/google`)
- **Headings:** Playfair Display (serif)
- **Body:** Inter (sans-serif)

### Print CSS
- `@media print` rules hide: Header, Footer, Navigation, Print button, Theme toggle
- `.print-only` elements are visible only in print
- Recipe card prints clean: logo, title, ingredients, instructions, nutrition, URL

## 8. Nutrition System

### Local Ingredient Database (Free, No API)

We use a **local nutrition calculator** — no API keys needed. A database of 150+ common ingredients with per-100g nutritional values.

### CLI Tool
```bash
npx tsx scripts/nutrition-calc.ts --servings 4 "2 cups flour, 1 tbsp sugar, 3 eggs"
```

Outputs YAML-ready frontmatter. Copy-paste into recipe `nutrition:` field.

### Files
- `src/lib/nutrition-db.ts` — Ingredient database (150+ items)
- `src/lib/nutrition-calc.ts` — Calculation engine (parses ingredient strings, estimates weights, computes totals)
- `src/lib/nutrition.ts` — Public API for programmatic use
- `scripts/nutrition-calc.ts` — CLI tool

### Limitations
- ~150 common ingredients — exotic items may not be found
- Volume-to-weight conversions are estimates
- No micronutrient data (vitamins/minerals)

## 9. Cloudflare Deployment Pipeline

### First-Time Setup
1. Create GitHub repo and push:
   ```bash
   git init
   git add .
   git commit -m "Initial scaffold"
   gh repo create appropriated-kitchen --public --source=. --remote=origin --push
   ```
2. In Cloudflare Dashboard → Pages → Connect Git → Select repo
3. Build settings:
   - Build command: `npx @opennextjs/cloudflare build`
   - Build output: `.vercel/output/static`
   - Environment variables: Add `EDAMAM_APP_ID`, `EDAMAM_APP_KEY`, `NEXT_PUBLIC_SITE_URL`
4. Add custom domain in Cloudflare Pages settings

### Local Preview (before deploy)
```bash
npm run pages:preview
```

## 12. Common Pitfalls to Avoid

| Pitfall | Why It Happens | How to Avoid |
|---------|---------------|--------------|
| `fs` crashes on Cloudflare | Using `readFileSync` in a Server Component without `generateStaticParams` | Always export `generateStaticParams` from any route that reads content files |
| `params` type error | Next.js 15+ made `params` a Promise | Always `const { slug } = await params;` |
| shadcn components don't render | Wrong shadcn style (old forwardRef vs new data-slot) | Use latest shadcn; `npx shadcn@latest init` with Radix Luma |
| Tailwind classes not applying | Config in `tailwind.config.*` instead of CSS `@theme` | All custom theme goes in `globals.css` |
| Excerpt shows `undefined` | Frontmatter uses `description:` instead of `excerpt:` | Always use `excerpt:` in MD/MDX frontmatter |
| NPM install fails | Package name has capital letters | Use lowercase: `"name": "appropriated-kitchen"` |
| Print includes nav/footer | No `@media print` hiding rules | Add `header nav footer .theme-toggle` to print:hidden |
